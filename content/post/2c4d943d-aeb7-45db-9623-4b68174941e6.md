默认的doom-emacs的配置肯定是不可能全部满足我们的需求的，因此我们需要在doom-emacs的配置上面再定制一些我们自己的分类。

<!--more-->

doom-emacs所有的配置是按照模块来进行分类的，我们可以在 `~/.doom.d/init.el` 里面看到下面的配置

```elisp
(doom! :completion
       company           ; the ultimate code completion backend
       ;;helm              ; the *other* search engine for love and life
       ;;ido               ; the other *other* search engine...
       ivy               ; a search engine for love and life

       :lang
       ;;agda              ; types of types of types of types...
       ;;cc                ; C/C++/Obj-C madness
       ;;clojure           ; java with a lisp
)
```

这里的 `:completion` 、 `:lang` 分别相当于两个大类，每个大类下面又分为一些小的分类，例如:

1.  `:completion` 子分类
    a. `company`
    b. `helm`
    c. `ido`
    d. `……`
2.  `:lang` 子分类
    a. `agda`
    b. `cc`
    c. `clojure`
    d. `……`


## 最简单的新建模块 {#最简单的新建模块}

在 `~/.doom.d/modules/` 下面创建对应目录就可以了，例如我这里想让一些基础的操作更加好用，因此我想创建一个 `better-defaults` 模块，分类我就命名成 `vinurs`
因此
`mkdir -p ~/.doom.d/modules/vinurs/better-defaults`
这个模块的相关配置就放在这个目录下面，最简单的模块我们只需要包含一个名为 `config.el` 的文件就可以了，例如在config.el里面增加一行代码 `(message "hello, this is better defaults module!")`
然后在 `~/.doom.d/init.el` 里面增加我们自己的模块

```elisp
(doom! :input
       ;;chinese
       ;;japanese
       ;;layout            ; auie,ctsrnm is the superior home row

       ;; 注意: 这是我们新增的模块
       :vinurs
       better-defaults
)
```

每次修改完了doom-emacs的配置我们都需要执行一下 `doom sync` ，然后重启一下emacs，这时候我们就会在 `*Message*` buffer里面看到到一行输出了 `hello, this is better defaults module!`
类似地，如果有别的配置，放在config.el里面就可以了。


## 文件加载顺序 {#文件加载顺序}

这个是一个完整的模块目录可能会包含的文件:

> vinurs/
>   better-defaults/
>     test/\*.el
>     autoload/\*.el
>     autoload.el
>     init.el
>     cli.el
>     config.el
>     packages.el
>     doctor.el

但是并不是所有的文件都是一定需要的。

这些文件的加载顺序如下

1.  config.el
2.  packages.el

完整的参考文档可以参考[官方文档](https://github.com/hlissner/doom-emacs/blob/develop/docs/getting%5Fstarted.org#writing-your-own-modules)


## 模块加载顺序 {#模块加载顺序}

doom-emacs里面这么多模块，那么这些模块是按照怎样的顺序来加载的呢？

1.  config.el
    lll

2.  packages.el

3.  ll

To create your own module you need only create a directory for it in ~/.doom.d/modules/abc/xyz, then add :abc xyz to your doom! block in ~/.doom.d/init.el to enable it.

In this example, :abc is called the category and xyz is the name of the module. Doom refers to modules in one of two formats: :abc xyz and abc/xyz.


## fork doom {#fork-doom}

既然选择了doom-emacs，因此我就决定先深入研究一下它，后面可能会进行魔改，因此我就先fork了一下doom-emacs，同时把它变成我的私有仓库。
在github上面新建一个空的私有仓库 `doom-emacs` ，然后执行下面的命令进行fork

```sh
# clone hlissner的doom-emacs
git clone --bare https://github.com/hlissner/doom-emacs.git
# 将上面clone的doom-emacs推送到我们新建的私有仓库
cd doom-emacs
git push --mirror https://github.com/vinurs/doom-emacs.git
# 重新clone我们自己的doom-emacs
git clone https://github.com/vinurs/doom-emacs.git ~/.emacs.d
```


### 增加代理 {#增加代理}

doom很多事情是通过 `doom` 这个命令来执行的，由于国内网络的原因，因此我们要配置上对应的代理，要不然有些包通过doom命令行是下载不下来的。

```elisp
(setq url-gateway-method 'socks)
(setq url-proxy-services
      '(("http"     . "127.0.0.1:7890")
        ("https"     . "127.0.0.1:7890")))
(setq socks-server '("Default server" "127.0.0.1" 7891 5))
```


## 初始化doom-emacs {#初始化doom-emacs}

```sh
~/.emacs.d/bin/doom install
```

为了以后方便使用doom，我们把doom的路径加到 `zsh` 配置文件里面去

```sh
export PATH="$HOME/.emacs.d/bin:$PATH"
```
