Emacs是神用的编辑器，但是神器要配上对应的配置才能发挥出它的威力，目前完整的emacs相关的配置是很多的，例如spacemacs、doom-emacs、Centaur Emacs……从零开始配置自己的Emacs我目前还没有那个实力，也不一定配置得就比别人好，因此我选择先站在巨人的肩膀上进行配置，等有实力自己配置并且现有的配置不能满足我的需求的时候再从头开始配置。

## fork doom {#fork-doom}

既然选择了doom-emacs，因此我就决定先深入研究一下它，后面可能会进行魔改，因此我就先fork了一下doom-emacs，同时把它变成我的私有仓库。
在github上面新建一个空的私有仓库 `doom-emacs` ，然后执行下面的命令进行fork

```sh
# clone hlissner的doom-emacs
git clone --bare https://github.com/hlissner/doom-emacs.git
# 将上面clone的doom-emacs推送到我们新建的私有仓库
cd doom-emacs
git push --mirror https://github.com/vinurs/doom-emacs.git
# 重新clone我们自己的doom-emacs
git clone https://github.com/vinurs/doom-emacs.git ~/.emacs.d
```


### 增加代理 {#增加代理}

doom很多事情是通过 `doom` 这个命令来执行的，由于国内网络的原因，因此我们要配置上对应的代理，要不然有些包通过doom命令行是下载不下来的。

```elisp
(setq url-gateway-method 'socks)
(setq url-proxy-services
      '(("http"     . "127.0.0.1:7890")
        ("https"     . "127.0.0.1:7890")))
(setq socks-server '("Default server" "127.0.0.1" 7891 5))
```


## 初始化doom-emacs {#初始化doom-emacs}

```sh
~/.emacs.d/bin/doom install
```

为了以后方便使用doom，我们把doom的路径加到 `zsh` 配置文件里面去

```sh
export PATH="$HOME/.emacs.d/bin:$PATH"
```

1.  增加代理
    由于国内网络环境的原因，很多包都无法正常下载，因此我们要配置上对应的代理，在 `~/.doom.d/config.el` 里面增加

